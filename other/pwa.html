1. Required File Types

To build a "real" PWA from scratch, you typically need 3 files:

  • index.html - The app interface and iOS meta tags.
  • manifest.json - The "Web App Manifest" for Android/Chrome.
  • icon.png (or .svg) - The app icon (at least 180x180px for iOS).

2. The HTML Syntax

Place these tags inside your <head> for iOS support:

index.html
<!-- Enable App Mode -->
<meta name="apple-mobile-web-app-capable" content="yes">

<!-- Status Bar Style (default, black, or black-translucent) -->
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">

<!-- App Title on Desktop -->
<meta name="apple-mobile-web-app-title" content="My Car App">

<!-- The Icon (PNG or SVG) -->
<link rel="apple-touch-icon" href="icon.png">

3. The Manifest File

Android and Chrome use a JSON file to define the app behavior.

manifest.json
{
  "name": "My Car App",
  "short_name": "CarApp",
  "start_url": "index.html",
  "display": "standalone",
  "background_color": "#ffffff",
  "theme_color": "#007AFF",
  "icons": [
    {
      "src": "icon.png",
      "sizes": "192x192",
      "type": "image/png"
    }
  ]
}

Link it in HTML: <link rel="manifest" href="manifest.json">
